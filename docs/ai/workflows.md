# Workflows

Use these workflows for common AI tasks.

## Fix bug

- Reproduce the bug when possible.
- Fix the root cause.
- Run local QA from `docs/ai/qa.md`.
- Follow `docs/ai/deploy-to-production.md` before publishing.

## Implement feature

- Implement the requested feature.
- Keep the change focused.
- Run local QA from `docs/ai/qa.md`.
- Follow `docs/ai/deploy-to-production.md` before publishing.

## Refactor

- Refactor without changing behavior.
- Run local QA from `docs/ai/qa.md`.
- Follow `docs/ai/deploy-to-production.md` before publishing.

## Review PR

- Review the change.
- Lead with concrete findings.
- Check `CHANGELOG.md`.
- Confirm QA was done.

## Write tests

- Add focused tests for the requested change.
- Run local QA from `docs/ai/qa.md`.
